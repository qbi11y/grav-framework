Components

Components are a collection of elements that represent a user interface task. Use multiple components to build out a page.

If you feel the provided components do not offer the support you are looking for, please send feedback so the current set of components can be addressed.

Page Action Bar

The page action bar contains page level actions that affect all items on the page. By default the page action bar allows a user to sort by, search, filter and perform actions as they relate to the content.

Expanded List

The expanded list component should be used to display data that should be shown in a list. You should consider using the expanded list component if there are several fields in the database that need to be displayed. This component allows for a lot of information to be shown while keeping the screen clean and readible.

Basic Table

The basic table component should be used to display tabular data. Avoid using the basic table component if the fields to be displayed are more than 7. If the fields to be displayed is greater than 7, consider using the expanded list component.

Filter

The filter component should be used when a large dataset is returned and it would be reasonable that a user would want to filter out data to get exactly what they are looking for. The filter should be triggered in the page action bar component with the filter window being displayed below the page action bar.
